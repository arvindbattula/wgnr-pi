# wgnr-pi Design Context

## Design Context

### Users
Solo power user (data scientist / PE professional) working at a desk in a dimly lit office.
Long focused sessions — coding, analysis, problem-solving. Not casual use. Eyes are on this UI for hours.

### Brand Personality
**Focused. Sharp. Technical.**
Not a consumer chat app. Closer to a professional tool — like a well-tuned IDE or terminal.
The interface should get out of the way and let the work dominate. No decorative noise.

### Aesthetic Direction
Reference: Claude Code TUI — monospace structure, clear contrast hierarchy, minimal chrome.
Light mode. Not "white and friendly" — more like a well-lit technical document.
Off-white/warm paper background. Dark ink for text. One restrained accent color.
Think: a Bloomberg terminal printed on paper, or a technical spec document that's been designed.

**Anti-references**: Dark navy with glowing blue accents (current). Consumer chat apps (ChatGPT, Gemini). 
Anything that looks like it came from a startup UI kit.

### Theme
Light — warm off-white background, not stark white. Dark near-black text for maximum contrast.
Accent: one color, used sparingly. Leaning toward a muted teal or slate — not bright blue, not purple.

### Design Principles
1. **Contrast first** — every text element must be immediately readable without effort
2. **Chrome recedes, content dominates** — UI elements should be quiet; messages and code are the focus
3. **Density without clutter** — pack information tightly but give each element room to breathe
4. **No decorative elements** — every pixel must earn its place; nothing purely ornamental
5. **Stats are data, not decoration** — token usage, cost, context % must be readable at a glance
